WHAT WAS IMPLEMENTED

File: src/tag_hover_sim/tag_hover_sim/hover_yaw_search_sensor_lock.py (431 lines)

3-Phase Supervisory State Machine:

1. SEARCH (Yaw spinning)
   → Waits for AprilTag detection via TF transform

2. ALIGN (Phase 1: Continuous P-Control)
   → Aggressive alignment: yaw, distance, lateral, vertical
   → Publishing velocity commands at 20 Hz
   → Monitors bounding box (±0.25m lateral, ±0.30m distance, ±0.08rad yaw)
   → Exits when tag stable in box for 2.0 seconds

3. HOVER_BOX (Phase 2: Event-Based Correction)
   → Zero velocity when tag inside box (allows settling)
   → P-control when tag outside box (brings back in)
   → Waits 2.0 more seconds before advancing

4. SENSOR_HOVER (Phase 3: Silent Handoff)
   → NO velocity commands published
   → FCU holds position via optical flow + rangefinder + EKF
   → Indefinite duration (until tag lost)
   → Supervisor role only

Tag Loss Fallback: Any phase → SEARCH immediately

CONFIGURABLE PARAMETERS
================================================================================

Phase 1 Control Gains:
  - lock_k_yaw: 0.1 (P gain for yaw)
  - lock_k_distance: 0.2 (P gain forward/back)
  - lock_k_lateral: 0.1 (P gain left/right)
  - lock_k_vertical: 0.1 (P gain up/down)
  - target_distance: 2.0 m (desired standoff)

Phase 2 Bounding Box:
  - lateral_box_m: 0.25 m (±left/right tolerance)
  - distance_box_m: 0.30 m (±forward/back tolerance)
  - yaw_box_rad: 0.08 rad (±yaw tolerance ~4.6°)

Timing:
  - equilibrium_time_s: 2.0 s (dwell per transition)
  - rate_hz: 20.0 Hz (control loop frequency)
  - search_yaw: 0.25 rad/s (yaw rate in SEARCH)

Tune at runtime:
  ros2 param set /hover_yaw_search lock_k_yaw 0.08
